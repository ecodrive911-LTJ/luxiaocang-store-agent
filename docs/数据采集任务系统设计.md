## 一、任务类型

### 类型A：对标商品比价（日常高频）

| 项目 | 内容 |
|------|------|
| **频率** | 每日或隔日 |
| **对象** | 30-50个核心对标SKU |
| **执行人** | 店员在手机上操作 |
| **方式** | 打开美团App搜索指定商品→浏览竞品价格→录屏 |
| **时长** | 每条录屏30-90秒（批量滑屏浏览） |
| **上传** | 录屏结束即上传 |
| **AI处理** | 抽帧→OCR识别商品名+价格→匹配对标商品→写入比价表 |

### 类型B：竞品全店扫描（周期深度）

| 项目 | 内容 |
|------|------|
| **频率** | 每1-2周/竞品店 |
| **对象** | 周边3-5家竞品便利店 |
| **执行人** | 店员到店 |
| **方式** | 在竞品店内用手机拍摄货架全景→逐品类录屏 |
| **时长** | 每条3-5分钟 |
| **AI处理** | 抽帧→品类识别→价格提取→生成竞品品类-价格矩阵 |

### 类型C：门店自检（内部运营）

| 项目 | 内容 |
|------|------|
| **频率** | 每日 |
| **对象** | 自家门店 |
| **方式** | 拍摄缺货货架→上报报损商品→盘点临期品 |
| **时长** | 每条30秒以内 |

## 二、任务下发流程

### 第一步：管理员在Web后台创建任务

```
Web后台 → 任务管理页面:
  ├─ 选择任务类型（比价/竞品扫描/自检）
  ├─ 选择对标商品清单（从商品库勾选）
  ├─ 选择目标竞品店（从竞品库勾选）
  ├─ 指派执行人（选店员）
  ├─ 设置截止时间
  └─ 点击"下发任务"
```

### 第二步：店员收到任务通知

店员通过两种方式接收：

**方式1：手机浏览器打开店参谋页面**（无需安装App）
- 登录自己的账号（store_owner角色）
- 首页显示"待执行任务"列表
- 点击任务查看详情：要采集哪些商品、哪个竞品店

**方式2：微信/企业微信通知**（后续开发）
- 收到消息"你有新的比价任务，共15个商品，截止今晚8点"

### 第三步：店员执行采集

店员在手机上：
1. 打开店参谋页面 → 点击"开始采集"
2. 页面显示本条任务要采集的商品列表
3. 切换到美团App（或竞品店内）
4. 按列表逐个搜索/浏览商品
5. **全程录屏**（手机自带录屏功能，或店参谋页面内嵌录屏）
6. 录屏结束 → 回到店参谋页面 → 点击"上传"

### 第四步：上传到服务器

```
店员手机 → HTTPS POST /api/tasks/{task_id}/upload
  ├─ 视频文件（MP4, ≤5分钟, ≤50MB）
  ├─ 关联任务ID
  └─ 自动标记为"已提交"
```

## 三、上传方式设计

### 方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **A. 浏览器直接上传** | 无需安装任何东西 | 视频大会卡、需稳定网络 | ⭐⭐⭐ 首选 |
| **B. 微信发送视频** | 店员最熟悉 | 微信压缩画质、AI识别率下降 | ⭐⭐ 备用 |
| **C. 企业网盘** | 大文件友好 | 多一步操作、体验割裂 | ⭐ 复杂 |
| **D. 店参谋内嵌录屏** | 一站式体验 | 开发成本高 | ⭐⭐⭐ 长期目标 |

### 推荐：A方案（浏览器直接上传）+ D方案作为长期演进

**当前实现**：店员用手机自带录屏功能录美团App操作 → 录完回到店参谋Web页面 → 点击上传按钮 → 选择相册中的视频 → 上传到服务器。

**上传接口设计**：
```
POST /api/tasks/{task_id}/items/{item_id}/upload
Content-Type: multipart/form-data
Body: video_file (MP4)
Response: { "ok": true, "status": "pending_analysis" }
```

## 四、任务数据结构

### tasks 表（任务主表）

```sql
CREATE TABLE tasks (
    id TEXT PRIMARY KEY,
    store_id TEXT NOT NULL,          -- 所属门店
    created_by TEXT NOT NULL,        -- 创建人(admin/manager)
    assigned_to TEXT NOT NULL,       -- 执行人(店员)
    task_type TEXT NOT NULL,         -- 'price_compare' | 'competitor_scan' | 'self_check'
    title TEXT NOT NULL,             -- 任务标题
    description TEXT,                -- 任务说明
    competitor_store_id TEXT,        -- 目标竞品店(竞品扫描时)
    status TEXT DEFAULT 'pending',   -- pending/assigned/in_progress/submitted/done/expired
    deadline TEXT,                   -- 截止时间
    created_at TEXT,
    updated_at TEXT
);
```

### task_items 表（任务明细 — 每个要采集的商品）

```sql
CREATE TABLE task_items (
    id TEXT PRIMARY KEY,
    task_id TEXT NOT NULL,
    product_name TEXT NOT NULL,      -- 要采集的商品名
    product_spec TEXT,               -- 规格（如500ml）
    product_barcode TEXT,            -- 条码（可选）
    category TEXT,                   -- 品类
    status TEXT DEFAULT 'pending',   -- pending/done/skipped
    sort_order INTEGER,              -- 排序（店员按顺序采集）
    FOREIGN KEY (task_id) REFERENCES tasks(id)
);
```

### uploads 表（上传记录）

```sql
CREATE TABLE uploads (
    id TEXT PRIMARY KEY,
    task_id TEXT NOT NULL,
    task_item_id TEXT,
    uploaded_by TEXT NOT NULL,
    filename TEXT NOT NULL,
    file_path TEXT NOT NULL,         -- 服务器存储路径
    file_size INTEGER,
    duration_seconds REAL,           -- 视频时长
    status TEXT DEFAULT 'pending',   -- pending/analyzing/done/failed
    result_json TEXT,                -- AI分析结果(JSON)
    error_message TEXT,
    uploaded_at TEXT,
    analyzed_at TEXT
);
```

## 五、AI分析流程

### 视频到达服务器后的处理链

```
视频上传 → uploads 表 status=pending
     │
     ▼
video_scheduler.py 轮询检测 (每10秒)
     │
     ▼
qwen_analyzer.py 分析:
  ├─ 步骤1: 用OpenCV抽关键帧（8帧均匀采样）
  ├─ 步骤2: 逐帧调用千问视觉模型 (qwen-vl-max)
  ├─ 步骤3: Prompt引导提取商品名+价格+店铺名
  └─ 步骤4: 返回 JSON 结果
     │
     ▼
结果写入:
  ├─ uploads.status → 'done'
  ├─ uploads.result_json → AI原始输出
  ├─ price_data 表 → 格式化后的比价记录
  │   (product_name, price, store_name, captured_at, source_video)
  └─ task_items.status → 'done'
     │
     ▼
Web后台可查看:
  ├─ 任务完成状态
  ├─ 比价结果表格
  └─ 竞品价格变化趋势图
```

### 比价专用的 Prompt 设计

```
你是便利店商品价格采集专家。分析此视频画面，提取商品信息。

视频来源：店员在美团App中搜索指定商品并浏览竞品店铺定价。

请提取所有可见商品，格式：
[
  {
    "product_name": "商品名称（必须和任务清单中的名称对应）",
    "spec": "规格（如500ml、250g）",
    "price": 数字价格,
    "store_name": "店铺名称",
    "has_promotion": true/false,
    "promotion_desc": "促销说明（如有）"
  }
]

要求：
1. 价格必须是数字，不要货币符号
2. 如果商品名无法匹配任务清单，标为 uncertain
3. 不要编造没看到的商品
```

## 八、关键设计决策

| 问题 | 决策 | 理由 |
|------|------|------|
| 店员如何上传？ | 手机浏览器直接上传MP4 | 零安装，任何手机都行 |
| 录屏工具？ | 手机自带录屏 | 不自己造轮子，保持简单 |
| 视频大小限制？ | ≤5分钟、≤50MB/条 | 手机录屏5分钟约30-50MB |
| AI分析时机？ | 上传后异步处理 | 不阻塞店员操作 |
| 分析失败怎么办？ | 自动重试2次，失败通知管理员 | 避免静默丢失数据 |
| 店员能看到别人的数据吗？ | 不能，store_owner只看到自己门店 | 已有权限体系支持 |
