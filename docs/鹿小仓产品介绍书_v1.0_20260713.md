## 前言：为什么便利店需要一个 AI 参谋？

我干便利店这行十几年了。看过太多这样的场景——

一个店主，凌晨两点还在店里理货。白天进货、看店、应付美团订单、跟供应商讨价还价，晚上对着账本发呆。他知道隔壁新开的罗森在抢他生意，但他没时间去美团上一个一个比价。他知道有些货三个月没动过了，但他没精力坐下来算动销率。他知道应该调价了、该引进新品类了，但"知道"和"做到"之间，隔着一千个琐碎的日常。

中国有超过30万家便利店。绝大部分是夫妻店、个体店、小型连锁。这些店养活了几百万个家庭，但在数字化这件事上，它们几乎是裸奔的。

不是不想数字化。是用不起。一套像样的零售ERP系统，动辄几万块。一个专职的数据分析师，月薪过万。请不起。学不会。用不上。

这就是鹿小仓·店参谋诞生的原因——

**我们不做一个需要你去学的系统。我们做一个能跟你对话的参谋。**

## 第一部分：鹿小仓·店参谋是什么？

### 一句话定义

**鹿小仓·店参谋是一款面向便利店/即时零售门店的 AI 经营决策 Agent。它不是一个工具、不是一套 SaaS、不是一个报表系统——它是一个能对话、能干活、能出主意的数字化合伙人。**

### 核心设计理念

**"像老板跟参谋聊天一样管理一家店。"**

你不需要学习任何操作界面。你只需要打开网页（或用手机浏览器打开），跟它说话：

- "帮我看看广安店这周哪些商品该调价了"
- "跟隔壁小柴购比一下饮用水品类的价格"
- "给我出一份财富店的选品优化方案"
- "派一个任务：让广安店店员去美团录一下竞品的饮料价格"

它理解你说的话，调用背后的数据、算法和自动化工具，给你可落地的答案。

### 产品本质：四个维度的融合

| 维度 | 你获得的能力 |
|------|------------|
| **数据中台** | 自有门店 + 竞品门店 + 商圈 POI，全部商品数据自动归集、结构化存储 |
| **分析引擎** | 品类结构诊断、价格竞争力分析、动销/滞销识别、毛利贡献拆解 |
| **决策建议** | 不是"你的毛利率低了"，而是"农夫山泉550ml建议从2.5元调到2.8元，理由是竞品均价2.6元、你进价涨了0.2元、该品在引流品中毛利偏低" |
| **执行系统** | 任务下发→店员采集→AI分析→结果入库→效果追踪，完整闭环 |

## 第三部分：门店全生命周期支持

### 阶段一：开店前（模块 0-2）—— "我该不该在这里开店？怎么开？"

**选址评估**：输入候选地址，鹿小仓调用高德 API 扫描周边 1-3 公里——
- 有多少小区？住了多少人？什么年龄段？
- 有多少写字楼？多少学校？多少竞品门店？
- 美团/饿了么这个区域的即时零售热度怎么样？
- 同类门店的客单价区间、热销品类是什么？

**建店规划**：确定要开店后——
- 根据面积倒推 SKU 总量上限
- 根据商圈人群画像确定品类权重（社区店多生鲜日配，写字楼店多即食快餐）
- 货架分配方案、冷链设备清单
- 首批进货清单（参考同商圈成功门店的品类结构）

**数据采集打底**：建店阶段就开始采集周边竞品数据，建立初始情报库。

### 阶段二：运营中（模块 3-8）—— "怎么让这家店赚钱？"

这是鹿小仓最核心的战场。

**选品规划（模块3）**
- 基于竞品热销数据，识别"你有我没有"的缺失品类
- 基于自身动销数据，识别滞销品建议淘汰
- 基于购物篮关联分析，配置连带组合（买泡面的人还买了什么？）
- 四层商品分层：引流品 / 利润品 / 常规品 / 长尾品，每层有不同管理逻辑

**定价策略（模块4）**
- 自动抓取美团/饿了么上竞品的同款商品价格
- 生成价格竞争力矩阵（你比竞品贵多少？便宜多少？）
- 智能调价建议：不是一刀切打折，而是——
  - 引流品：对标竞品底价，牺牲毛利换流量
  - 利润品：在竞品价格区间上沿定价，守护毛利
  - 常规品：跟随市场均价，保持竞争力即可
- 调价后追踪：调了价，销量变了吗？毛利变了吗？

**供应链（模块5）**
- 库存周转天数监控 → 预警囤货过多
- 缺货频次统计 → 预警备货不足
- 供应商比价 → 同款商品不同供应商的价格差异
- 临期商品预警 → 促销优先清单

**运营诊断（模块7）**
- 品类贡献度分析：哪些品类贡献了最多毛利？
- ABC 分类：A类商品（前20%贡献80%销售额）重点管理
- 交叉比率分析：毛利率 × 周转率，找到真正"好卖又赚钱"的商品
- 异常检测：销售额突然下降/上升的商品，需要人工关注

**促销营销（模块8）**
- 基于滞销品清单生成促销方案
- 基于节假日/天气/商圈活动推荐促销时机
- 促销效果追踪：活动期间增量 vs 活动成本，算 ROI

### 阶段三：扩张期（模块 6 + 9）—— "怎么从一家店变成五家店？"

**品牌管理（模块6）**
- 多门店统一选品标准
- 同一品牌不同门店的差异化策略对比
- 客户评价与口碑监控

**财务管理（模块9）**
- 单店损益表自动生成
- 多门店横向对比（谁赚得多？为什么？）
- 现金流预警
- 投资回报周期测算

## 第四部分：AI 与店主的关系——不是工具，是合伙人

这是鹿小仓跟市面上所有"零售管理系统"最大的不同。

### 传统系统：它是你的工具。你学会用它。

你打开系统 → 在菜单里找功能 → 点按钮 → 填表单 → 导出报表 → 自己看 → 自己做决定。

**你是操作者，系统是被操作者。**

### 鹿小仓：它是你的参谋。它学会帮你。

你说话 → 它理解你的意图 → 它调用背后的工具和数据 → 它给你分析结果和建议 → 你拍板。

**你是决策者，它是智囊团。**

这种关系更接近一个便利店老板和他信任的店长之间的关系：

- 老板说"最近生意不太好"，店长不会回一句"那您多努力"。店长会说"我查了一下，这周饮料品类销量下降了15%，主要是可口可乐涨价后顾客转去买百事了。我建议把可乐从3.5调到3.0，先拉回客流。"

- 鹿小仓做的就是这样的事。只不过它背后不是一个店长的经验，而是：
  - 📈 **消费品分析师**：懂品类结构、懂消费趋势
  - 💰 **投行专家**：会算投资回报、会做财务建模
  - 🚚 **供应链领导人**：懂库存周转、懂供应商管理
  - 🏪 **便利店连锁高管**：管过几十家店，知道什么坑不能踩

这四个人组成的"专家团队"，构成了鹿小仓的分析大脑。

### 三层权限：不同角色，看到不同的鹿小仓

| 角色 | 权限范围 | 典型使用者 |
|------|---------|-----------|
| **管理员 (admin)** | 全权限：看所有门店、管用户、改配置 | 品牌创始人/CEO |
| **总部管理 (manager)** | 看数据、下任务、做分析，不能改系统配置和删用户 | 运营总监/区域经理 |
| **分店店主 (store_owner)** | 只看自己的门店，只能录自己的数据 | 单店老板/加盟商 |

这种设计让鹿小仓既能服务单店，也能支撑连锁扩张。

## 第五部分：技术架构——轻量但扎实

### 部署方式
- 阿里云 ECS 服务器，无需本地安装
- 打开浏览器就能用，电脑手机都行
- 数据存储在你自己服务器上，不上传任何第三方平台

### 技术栈
- **后端**：Python FastAPI（轻量、高效、异步）
- **数据库**：SQLite（无需额外数据库服务，但支持完整的 SQL 查询和多表关联）
- **大模型**：火山引擎（支持流式对话，回复逐字生成，体验流畅）
- **认证**：bcrypt 密码哈希 + Session Token（安全性对标企业级应用）
- **采集引擎**：Playwright 浏览器自动化（自动抓取美团/饿了么页面数据）
- **地图服务**：高德 API（商圈 POI 扫描、周边人群画像）

### 安全设计
- 密码不存明文，使用 bcrypt 哈希
- API Key 不存在前端，后端代理调用
- 多租户数据隔离（单数据库 + user_id/store_id 字段级权限）
- 数据归属明确：每个查询自动带上用户身份，不会看到别人家的数据

## 第六部分：实际使用场景

### 场景一：每周例行运营诊断

> 店主打开鹿小仓，说："帮我做一下广安店这周的运营诊断。"

鹿小仓自动执行：
1. 调取广安店最新库存数据
2. 扫描周边竞品（小柴购、厉臣超市）的最新价格
3. 比价分析，找出价格劣势商品
4. 动销分析，找出滞销品
5. 品类贡献分析，找出利润黑洞
6. 输出一份带具体行动建议的诊断报告

**店主不用打开任何 Excel，不用手动比价，不用算公式。**

### 场景二：数据采集任务

> 管理员发现美团上某个竞品上新了一批网红商品，需要采集价格。

1. 在鹿小仓创建任务："竞品比价-饮料品类"，选择目标门店
2. 勾选需要采集的商品清单（从对标商品库中选）
3. 设置截止时间
4. 店员在手机端收到任务，去美团搜索录屏
5. 店员在鹿小仓聊天窗口上传视频
6. 系统自动调用 Qwen 视觉模型分析视频 → 提取商品名、规格、价格 → 入库
7. 管理员在任务详情看结果：哪些商品比竞品贵、哪些持平、哪些便宜

**一条任务串起：决策→执行→分析→入库。不需要微信群转发、Excel 手动录入。**

### 场景三：开店选址

> 想在承德双滦区再开一家店，哪里最合适？

鹿小仓：
1. 扫描承德双滦区所有小区、写字楼、学校、商业体
2. 统计各区域人口密度、年龄结构
3. 列出已有竞品分布（避免正面硬刚）
4. 输出推荐点位 + 每个点位的优劣势分析
5. 预估首年营收（基于同城同类型门店数据外推）

**这些事，以前需要一个选址顾问团队花几周时间调研。现在，一场对话。**

## 第八部分：给使用者的几句话

**给单店老板：**

你不需要数据分析师。你不需要学任何新系统。你只需要每天打开鹿小仓，跟它聊聊你的店。它会告诉你什么该调价了、什么该下架了、隔壁店在搞什么促销、你的钱花在了哪里。它不会替代你做决定，但它会让你做的每一个决定都有数据支撑，而不是拍脑袋。

**给连锁品牌管理者：**

你的每家店水平参差不齐，好的店长能把坪效做到行业前 10%，差的店长连库存都盘不清楚。鹿小仓给你一把尺子——所有门店的关键指标横向对比，优秀经验可复制，落后门店可诊断。你不需要飞遍所有城市巡店，数据会告诉你哪家店需要你关注。

**给想开店的创业者：**

开店最大的成本不是房租和装修，是"开错了"。选址错了，品类结构错了，定价策略错了——这些错误每一个都值几万甚至几十万。鹿小仓在你的决策之前，给你一个基于数据的推演。它不是算命，但它能让你看到盲区。

## 结语

便利店这门生意，说到底是三件事：

1. **知道卖什么**（选品）
2. **知道卖多少钱**（定价）  
3. **知道怎么卖得更好**（运营）

这三件事，每件事背后都需要数据、经验、判断力。过去，这些能力只存在于经营多年的老手脑子里。现在，鹿小仓把它们变成了一款产品。

它不是要代替店主的经验和直觉。它是给经验和直觉配上一个"数字大脑"。

你在前面经营。它在后面算账。你在台前决策。它在幕后做功。

这就是鹿小仓·店参谋。
