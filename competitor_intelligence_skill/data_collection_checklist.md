# 竞品数据采集清单（操作SOP）

## 采集频率建议
- **每日采集**：价格变动、活动变动、销量TOP20
- **每周采集**：全量SKU、新品上架、下架商品
- **每月采集**：店铺评分、月销量、配送策略变化

## 一、店铺基础信息（必采）

| 字段 | 说明 | 示例 |
|------|------|------|
| store_name | 店铺名称 | XX便利店(朝阳路店) |
| platform | 所在平台 | 美团/饿了么/抖音 |
| monthly_sales | 月销量(平台显示) | 5280单 |
| rating | 店铺评分 | 4.8分 |
| min_order_amount | 起送价 | 20元 |
| base_delivery_fee | 基础配送费 | 4元 |
| delivery_time | 预计配送时长 | 30分钟 |
| business_hours | 营业时段 | 07:00-02:00 |

## 二、商品数据（必采 - 全量SKU）

| 字段 | 说明 | 示例 |
|------|------|------|
| sku_name | 商品全称 | 可口可乐碳酸饮料330ml罐装 |
| category | 所属分类 | 饮料/碳酸饮料 |
| original_price | 原价(划线价) | 3.5元 |
| sell_price | 当前售价 | 3.0元 |
| activity_price | 活动价(如有) | 2.5元 |
| monthly_sales | 月销量 | 520件 |
| stock_status | 库存状态 | 有货/缺货 |
| is_new | 是否新品标识 | 是/否 |
| main_image_url | 主图链接 | (用于图像匹配备选) |

## 三、营销活动数据（必采）

| 活动类型 | 采集内容 | 示例 |
|----------|----------|------|
| 满减活动 | 所有满减档位 | 满39减5, 满69减12 |
| 折扣活动 | 折扣力度+适用品类 | 饮料品类8折 |
| 第N件优惠 | 第几件几折 | 第2件半价 |
| 限时秒杀 | 秒杀价+时段+商品 | 10:00-12:00 可乐1元 |
| 新客优惠 | 新客立减金额 | 新客立减8元 |
| 运费减免 | 免运费门槛 | 满49免配送费 |
| 赠品活动 | 赠品内容+条件 | 满59送湿巾 |

## 四、流量与排名数据（选采 - 高价值）

| 字段 | 说明 |
|------|------|
| search_rank | 搜索"便利店"时的排名位置 |
| category_rank | 品类排名 |
| has_ad_label | 是否有广告标识(竞价排名) |
| homepage_banner | 是否在首页有banner位 |
| popup_promotion | 进店弹窗活动内容 |

## 五、配送策略细节（选采）

| 字段 | 说明 |
|------|------|
| delivery_distance_max | 最大配送距离 |
| delivery_fee_tiers | 阶梯配送费规则 |
| express_delivery | 是否支持极速达及加价 |
| self_pickup_discount | 自提优惠 |

## 采集工具建议
1. **手动采集**：适合初期1-3家竞品，用Excel逐项记录
2. **RPA工具**：影刀/八爪鱼等，设置定时任务自动采集
3. **API接口**：如平台开放数据接口，优先使用API
4. **第三方数据服务**：如鲸参谋、多多情报通等O2O数据平台
