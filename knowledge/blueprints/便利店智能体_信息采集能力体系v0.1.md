## 一、我们需要采集什么信息？

### 给出一个地址，我们需要知道：

| 信息维度 | 具体内容 | 为什么需要 |
|---|---|---|
| **周边人口** | 人口密度、年龄结构、收入水平、职业分布 | 决定商圈类型和消费力 |
| **周边竞品** | 便利店/超市/小卖部的数量、品牌、距离、规模 | 判断竞争饱和度 |
| **交通设施** | 公交站、地铁站、停车场、主干道 | 评估客流可达性 |
| **商圈业态** | 住宅小区、写字楼、学校、医院、商业街 | 判断客流类型和消费场景 |
| **租金成本** | 区域商铺租金水平、转让费 | 测算投资回报 |
| **周边商户** | 餐饮、药店、快递点等互补业态 | 判断商圈成熟度 |

### 给出一个品类/商品，我们需要知道：

| 信息维度 | 具体内容 | 为什么需要 |
|---|---|---|
| **批发价格** | 1688/义乌/各地批发市场供货价 | 测算毛利、找最低采购成本 |
| **零售价格** | 美团/饿了么/京东到家上的同品价格 | 竞争导向定价参考 |
| **品牌格局** | 这个品类有哪些主流品牌、市场占有率 | 选品决策 |
| **趋势热度** | 社交媒体讨论度、搜索趋势 | 判断上升/衰退品类 |
| **供应商类型** | 品牌直供/经销商/批发市场/一件代发 | 供应链搭建决策 |

### 给出一个门店的经营数据，我们需要知道：

| 信息维度 | 具体内容 | 为什么需要 |
|---|---|---|
| **自身销售** | 品类销售、单品销售、毛利率、周转率 | 运营诊断 |
| **竞品动态** | 周边竞品的价格变化、新品上架、促销活动 | 竞争响应 |
| **行业趋势** | 便利店行业新品趋势、季节性品类变化 | 选品更新 |

## 二、我们能用什么手段获取？—— 能力清单

### 🟢 能力A：地图API POI搜索（立即可用）

**工具**：高德地图开放平台 / 百度地图开放平台
**获取方式**：免费注册开发者账号，获取API Key，调用REST接口
**免费配额**：
- 高德：个人开发者每日5000次，并发50/秒
- 百度：个人开发者地点检索100次/日（较少），企业试用版2000次/日

**能采集什么**：
- 周边便利店/超市/商场 POI列表（名称、地址、坐标、距离）
- 周边公交站、地铁站、停车场
- 周边住宅小区、写字楼、学校、医院
- 周边餐饮、药店等互补业态
- 逆地理编码（坐标→地址描述）

**限制**：
- 无法获取人口密度/收入水平（需要百度慧眼等付费产品）
- 无法获取竞品的商品价格和SKU结构

**实施步骤**：
1. 注册高德开放平台开发者账号 → 获取Web服务API Key
2. 封装Python模块：`poi_search(location, radius, type) → list[POI]`
3. 给定地址 → 地理编码获取坐标 → 周边POI搜索 → 生成商圈报告

---

### 🟢 能力B：Web搜索 + 网页抓取（已有，需增强）

**工具**：web_search + web_fetch + browser自动化
**获取方式**：当前已具备

**能采集什么**：
- 行业趋势报告、消费趋势新闻
- 区域租金水平（通过房产网站搜索）
- 便利店行业经营数据（公开报告）
- 社交媒体消费趋势（抖音/小红书热门品类）
- 7-11/全家/罗森等品牌的新品动态

**限制**：
- 搜索结果泛化，需要结构化提取
- 无法获取实时精确价格
- 需要定期执行才能保持时效性

**增强方向**：
- 建立关键词库：便利店+选品、便利店+趋势、便利店+毛利、承德+商圈等
- 定时执行（cron）：每周自动搜索行业新闻，存入知识库
- 结构化提取：不是只存链接，而是提取关键数据点存入知识库

---

### 🟡 能力C：线上零售平台商品信息采集（需要开发）

**目标平台**：美团外卖、饿了么、京东到家、多多买菜

**能采集什么**：
- 周边便利店的在线商品列表和价格
- 竞品的SKU结构、价格体系
- 促销活动信息
- 月销量数据（部分平台）

**技术路径**：
1. **浏览器自动化（xbrowser skill）**：控制真实Chrome浏览器，登录美团/饿了么，按位置搜索周边便利店，抓取商品列表和价格
2. **第三方数据服务**：鼎点数据等专业采集工具（付费）
3. **API接口**：部分平台有开放API（需申请）

**推荐路径**：用xbrowser skill做浏览器自动化采集，这是最可控的方式。

**限制**：
- 需要登录态（用xbrowser的login-state reuse）
- 反爬机制（需要模拟真人行为，控制频率）
- 数据时效性（需定期更新）

**实施步骤**：
1. 用xbrowser打开美团外卖 → 定位到目标地址 → 搜索"便利店" → 逐个进入店铺 → 抓取商品名称、价格、分类
2. 输出标准化Excel：店铺名、商品名、价格、分类、月销量
3. 封装为独立模块：`competitor_price_scan(address) → Excel`

---

### 🟡 能力D：线上批发市场价格采集（需要开发）

**目标平台**：1688（阿里巴巴）、义乌购、拼多多批发

**能采集什么**：
- 指定商品的批发价格区间
- 供应商列表和资质
- 起订量、物流费率
- 热销商品排行

**技术路径**：
1. **1688搜索**：用xbrowser或web_fetch搜索指定商品关键词 → 提取价格区间、供应商信息
2. **1688 API**：有第三方封装的1688商品详情API（可获取价格、销量、SKU等）
3. **新通路/零售通**：阿里零售通（B2B采购平台）有商品目录

**限制**：
- 1688有反爬机制，需要登录态
- 批发价格因采购量不同而变化
- 部分API为第三方封装，稳定性需验证

**实施步骤**：
1. 先用xbrowser做基础搜索：关键词 → 价格区间 → 供应商列表
2. 逐步积累品类基准价格库
3. 对比零售价格 → 计算毛利空间

### 🟡 能力E：周边人口与消费力数据（部分免费）

**数据源**：
1. **百度慧眼**：人口热力图、人口画像（部分城市免费，深圳/上海数据最完整）
2. **高德地图热力图**：实时人流热力（可视化，非数据接口）
3. **国家统计局/地方统计局**：区县级人口、收入、消费支出数据
4. **第七次人口普查数据**：街道级别人口数据（公开可查）
5. **顺为城市地图等第三方平台**：付费但有详细的人口画像

**能采集什么**：
- 区域人口密度（免费，精度到区县/街道）
- 人口年龄结构（免费，精度到区县）
- 平均收入水平（免费，精度到区县）
- 实时人流热力（百度慧眼，部分城市免费）
- 消费支出结构（统计年鉴，年度数据）

**限制**：
- 免费数据精度只能到区县/街道，不到小区级别
- 实时人流热力只有部分城市开放
- 收入数据是平均值，不反映分布

**实施步骤**：
1. 先用免费数据建基础：区县级人口+收入+消费支出
2. 百度慧眼API获取热力图（如果目标城市支持）
3. 付费数据作为增强（后期投入）

## 五、信息采集 → 决策输出的完整链路

```
用户输入："承德市南营子大街广安购物中心"

Step 1: 地理编码
  地址 → 高德API → 经纬度坐标

Step 2: 周边POI搜索（能力A）
  坐标 → 1km半径 → 搜索便利店/超市/商场/学校/写字楼/医院/公交站
  → 输出：周边业态分布表 + 竞品清单

Step 3: 竞品价格采集（能力C）
  竞品清单 → 美团/饿了么搜索 → 抓取各店商品价格
  → 输出：竞品价格对比表

Step 4: 人口消费力（能力E）
  坐标 → 所属区县 → 统计年鉴查询
  → 输出：人口密度 + 收入水平 + 消费结构

Step 4: 批发价格基准（能力D）
  竞品热销品类 → 1688搜索 → 批发价格
  → 输出：品类毛利空间测算

Step 5: 综合分析
  POI + 竞品价格 + 人口数据 + 批发价格 + 已有案例库
  → 输出：选址评分 + 建店方案 + 选品建议 + 定价策略
```

```python
class POIScanner:
    def __init__(self, amap_key):
        self.key = amap_key
        self.base_url = "https://restapi.amap.com/v3/place/around"
    
    def geocode(self, address):
        """地址 → 经纬度"""
        pass
    
    def search_nearby(self, location, radius, poi_type):
        """周边POI搜索"""
        # poi_type: 便利店=060101, 超市=060400, 学校=141200, 
        #           写字楼=120201, 医院=090100, 公交站=150700
        pass
    
    def scan_competitors(self, address, radius=1000):
        """一键竞品扫描"""
        coord = self.geocode(address)
        competitors = self.search_nearby(coord, radius, "060101|060400")
        return self._format_report(competitors)
    
    def scan_business_district(self, address, radius=1000):
        """一键商圈扫描"""
        coord = self.geocode(address)
        results = {}
        for name, poi_type in [
            ("便利店", "060101"), ("超市", "060400"),
            ("学校", "141200"), ("写字楼", "120201"),
            ("医院", "090100"), ("公交站", "150700"),
            ("地铁站", "150500"), ("住宅小区", "120300"),
        ]:
            results[name] = self.search_nearby(coord, radius, poi_type)
        return results
```
