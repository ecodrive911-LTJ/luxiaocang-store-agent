# 便利店商业智能体 — 架构蓝图 v0.1

> 「从开店第一天到关店最后一天，全程陪伴决策。建店方案不是一次性的，运营数据会反过来修正建店策略。」

---

## 一、核心定位

**不是一个出报告的工具，是一个会进化的经营决策中枢。**

它背后是一个虚拟专家团队：

| 专家角色 | 思考方式 | 在Agent中的职责 |
|---|---|---|
| 消费品行业分析师 | "这个品类在增长还是衰退？消费者在变什么？" | 行业趋势研究、消费洞察、品类生命周期判断 |
| 投行专家 | "这笔投入多久回本？现金流撑得住吗？" | 投资回报测算、资金规划、财务模型 |
| 供应链企业领导人 | "从工厂到货架最短路径是什么？库存周转几次？" | 采购体系设计、供应商管理、库存优化 |
| 便利店连锁经营高管 | "这个位置能不能开？开了怎么赚钱？赚不到怎么调？" | 选址、建店、选品、定价、运营的全链条决策 |

---

## 二、核心理念：建店与运营是一个闭环

```
建店阶段                          运营阶段
    │                                │
    ▼                                ▼
选址评估 ──→ 建店规划 ──→ 开业 ──→ 数据采集 ──→ 运营诊断
    ↑           │         │          │            │
    │           ↓         │          ↓            │
    │      选品规划       │      竞品监控          │
    │           │         │          │            │
    │           ↓         │          ↓            │
    │      定价策略       │      趋势洞察          │
    │           │         │          │            │
    └───────────┴─────────┴──────────┴────────────┘
                        反馈修正
```

**关键理解：建店方案不是"做完就结束"的交付物，而是"持续被运营数据修正"的活文档。**

- 选址时判断"这个商圈适合做日化为主"→ 运营3个月发现零食卖得更好 → 修正选品结构
- 建店时定"引流品占比10%"→ 运营发现引流品转化率低 → 修正引流策略
- 定价时按"成本加成30%"→ 竞品降价 → 修正定价模型

## 三、能力体系（9大模块，覆盖全生命周期）

### 第一阶段：建店前（Pre-Open）

#### 模块1：选址评估
**输入**：地址/坐标/商圈名称
**输出**：选址评分报告 + 开/不开建议

| 子能力 | 说明 | 数据来源 |
|---|---|---|
| 周边人口画像 | 人口密度、年龄结构、收入水平 | 地图API + 公开统计 |
| 竞品密度分析 | 周边便利店/超市数量、距离、品牌 | 地图POI搜索 |
| 交通便利度 | 公交站、地铁站、停车场 | 地图API |
| 商圈类型判断 | 住宅区/写字楼/学校/商业街/交通枢纽 | 地图POI分类 |
| 租金/成本预估 | 区域租金水平、转让费、装修成本 | 房产平台 + 行业经验 |
| 综合评分 | 5维度加权打分，给出明确建议 | 算法模型 |

**我们已有的数据基础**：承德南营子大街（广安购物中心）和财富广场两套商圈竞品数据。

#### 模块2：建店规划
**输入**：选址报告 + 资金预算
**输出**：建店执行方案

| 子能力 | 说明 |
|---|---|
| 门店规模建议 | 面积、货架数、冷柜数、收银台数 |
| 投资测算 | 装修+设备+首批进货+流动资金 = 总投资 |
| 工期规划 | 从签约到开业的时间节点 |
| 证照清单 | 营业执照、食品经营许可、烟草专卖等 |
| 设备采购清单 | 货架、冷柜、收银系统、监控、空调 |

#### 模块3：选品规划
**输入**：选址报告 + 门店规模 + 商圈类型
**输出**：首批SKU清单 + 品类结构方案

| 子能力 | 说明 |
|---|---|
| 品类结构推荐 | 根据商圈类型推荐品类比例 |
| 首批SKU清单 | 2000-3000个SKU的初始清单 |
| 引流品规划 | 每个品类15-20个引流品，总量200+ |
| 利润品筛选 | 高毛利商品作为利润引擎 |
| 季节性调整 | 根据季节/节日调整品类 |
| 竞品差异化选品 | 从竞品缺失品类中找机会（我们已有实战经验）|

**我们已有数据基础**：鹿小仓广安店2941 SKU + 财富店1386 SKU + 小柴购7659 SKU + 厉臣1996 SKU，可建立区域选品基准库。

#### 模块4：定价策略
**输入**：采购价 + 品类 + 竞品价格
**输出**：完整定价表 + 毛利结构方案

| 子能力 | 说明 |
|---|---|
| 成本加成定价 | 按品类设定目标毛利率 |
| 竞争导向定价 | 高敏感商品参考竞品 |
| 引流品定价 | ≤1元价格锚点 |
| 价格带规划 | 0-1元/1-5元/5-10元/10-20元/20-50元分布 |
| 毛利结构管理 | 分品类毛利目标 + 整体毛利测算 |

**我们已有数据基础**：广安店89.8%采购价覆盖率，可建立品类加价率基准。

---

### 第二阶段：建店中（Build）

#### 模块5：供应链搭建
**输入**：选品清单 + 门店位置
**输出**：供应商对接方案 + 采购体系设计

| 子能力 | 说明 |
|---|---|
| 供应商匹配 | 品牌直供/批发市场/经销商/一件代发的分层匹配 |
| 采购成本优化 | 多供应商比价、集中采购、账期谈判策略 |
| 物流配送方案 | 自采vs配送、频次、最低起订量 |
| 库存体系设计 | 安全库存、补货周期、周转目标 |
| 供应链数字化 | ERP/进销存系统选型建议 |
| 供应商评估体系 | 品质、价格、时效、退换货的评分标准 |

**核心理念**：供应链不是"找到货源就行"，而是"从工厂到货架最短路径+最低成本+最快周转"。

#### 模块6：品牌与视觉
**输入**：品牌定位 + 预算
**输出**：品牌视觉方案 + 空间设计建议

| 子能力 | 说明 |
|---|---|
| 品牌定位 | 目标客群画像 + 品牌人格 |
| 门头/招牌设计 | 视觉识别系统 |
| 店内动线设计 | 顾客行走路线 + 商品陈列逻辑 |
| 线上店铺视觉 | 外卖平台店铺装修风格 |

---

### 第三阶段：运营中（Operations）— 持续闭环

#### 模块7：运营诊断与优化
**输入**：门店销售数据 + 库存数据 + 竞品数据
**输出**：诊断报告 + 优化建议 + 执行清单

| 子能力 | 说明 |
|---|---|
| 毛利分析 | 品类毛利、单品毛利、负毛利预警 |
| 引流品效率 | 引流品占比、转化率、关联销售 |
| 库存健康度 | 周转率、缺货率、滞销品、低库存预警 |
| 品类平衡度 | 品类结构是否偏离标准模型 |
| 价格带分析 | 价格分布是否匹配客群 |
| 动态汰换建议 | 每周淘汰5%末位SKU的执行清单 |
| 竞品对比 | 与周边竞品的品类/价格/引流差异分析 |

**我们已有数据基础**：已完成广安店vs小柴购、财富店vs厉臣两套竞品分析方法论和脚本。

#### 模块8：促销与营销
**输入**：运营诊断 + 节日/季节/事件
**输出**：促销方案 + 活动日历

| 子能力 | 说明 |
|---|---|
| 促销策略 | 满减/买赠/限时折扣/会员积分 |
| 节日营销 | 节日提前备货+促销方案 |
| 线上引流 | 外卖平台活动、搜索优化 |
| 会员体系 | 社群运营、复购激励 |
| 内容营销 | 抖音/小红书探店、商品种草 |

#### 模块9：财务与资金管理
**输入**：经营数据 + 资金状况
**输出**：财务报告 + 资金规划

| 子能力 | 说明 |
|---|---|
| 经营仪表盘 | 日/周/月营收、毛利、坪效、人效 |
| 现金流管理 | 应收/应付、账期、资金周转 |
| 投资回报追踪 | 实际ROI vs 预期ROI 对比 |
| 扩张资金规划 | 单店盈利→多店复制的资金模型 |
| 融资建议 | 商户贷、供应链金融、融资租赁等工具 |

---

### 闭环机制：数据回流

```
运营数据 ──→ 诊断发现 ──→ 策略修正 ──→ 执行 ──→ 新数据
    ↑                                              │
    └──────────────────────────────────────────────┘
```

**具体实现**：
1. 每周/每月采集门店数据（销售、库存、竞品）
2. 与建店时的基准方案对比（预期 vs 实际）
3. 识别偏差 → 生成修正建议
4. 跟踪修正后的效果 → 更新基准
5. 积累的案例反哺知识库（下次建店更准）

## 八、与当前Agent的关系

当前Agent（luxiaocang）是"通用个人助手"，store-strategist是它体内的"专业便利店顾问"。

**调用方式**：
- 用户对luxiaocang说"帮我分析一下这个选址" → luxiaocang识别到便利店专业问题 → spawn store-strategist subagent执行
- 用户直接说"我要开店" → 自动进入建店流程
- 用户说"我的店引流品太少了" → 自动进入运营诊断流程

**数据共享**：
- store-strategist的知识库在独立workspace
- 但可以通过文件系统访问桌面的原始数据
- 分析结果写回知识库，不污染主workspace
