# 便利店运营智能体 — 能力蓝图与搭建方案

## 一、我是谁：这个智能体的定位

**角色**：资深便利店开店顾问 & 运营策略官
**服务对象**：便利店经营者（单店/连锁）、零售创业者
**核心价值**：把"开了10年店才知道的经验"变成可执行的数据驱动决策

---

## 二、能力矩阵：我需要具备什么能力

### 🔷 能力一：选址评估
**输入**：地址/坐标 → **输出**：选址评分报告

| 子能力 | 说明 | 数据来源 |
|--------|------|----------|
| 周边人口画像 | 人口密度、年龄结构、收入水平 | 百度/高德地图API、公开统计数据 |
| 竞品密度分析 | 周边便利店/超市数量、距离、品牌 | 地图搜索POI |
| 交通便利度 | 公交站、地铁站、停车场 | 地图API |
| 商圈类型判断 | 住宅区/写字楼/学校/商业街 | 地图POI分类 |
| 租金/成本预估 | 区域租金水平 | 房产平台数据 |
| 综合评分 | 加权打分，给出"开/不开/谨慎"建议 | 算法模型 |

### 🔷 能力二：选品规划
**输入**：门店信息（面积、商圈类型、目标客群）→ **输出**：SKU结构方案

| 子能力 | 说明 |
|--------|------|
| 品类结构推荐 | 根据商圈类型推荐品类比例（如住宅区 vs 写字楼差异） |
| 首批SKU清单 | 基于行业基准+区域偏好，生成2000-3000个SKU的初始清单 |
| 引流品筛选 | 识别高敏感低毛利商品作为引流品 |
| 利润品筛选 | 识别低敏感高毛利商品作为利润引擎 |
| 季节性调整 | 根据季节/节日调整品类比例 |
| 动态汰换规则 | "周分析月调整"机制，每周淘汰5%末位SKU |

### 🔷 能力三：定价策略
**输入**：采购价 + 品类 + 竞品价格 → **输出**：定价表

| 子能力 | 说明 |
|--------|------|
| 成本加成定价 | 基础定价法，按品类设定目标毛利 |
| 竞争导向定价 | 对高敏感商品参考竞品定价 |
| 心理定价 | 9.9/19.9等价格锚点 |
| 引流品定价 | ≤1元引流品策略 |
| 毛利率管理 | 分品类毛利率目标（鲜食40%+、饮料25%、日百20%） |
| 动态调价建议 | 基于库存周转/保质期/季节的调价信号 |

### 🔷 能力四：运营诊断
**输入**：门店销售数据/库存数据 → **输出**：诊断报告+优化建议

| 子能力 | 说明 |
|--------|------|
| 毛利分析 | 品类毛利、单品毛利、负毛利预警 |
| 引流品效率 | 引流品占比是否合理、是否带动关联销售 |
| 库存健康度 | 周转率、缺货率、滞销品 |
| 品类平衡度 | 品类结构是否偏离标准模型 |
| 价格带分析 | 价格分布是否匹配客群消费力 |

### 🔷 能力五：持续运营辅导
**输入**：定期数据更新 → **输出**：月度运营建议

| 子能力 | 说明 |
|--------|------|
| 月度经营复盘 | 自动生成月度报告 |
| 竞品价格监控 | 定期抓取周边竞品价格 |
| 趋势洞察 | 社交媒体趋势→选品建议 |
| 活动策划 | 节日促销/新品上架建议 |

## 五、核心算法设计

### 1. 选址评分模型
```
总分 = 人口密度(25%) + 竞品密度(20%) + 交通便利(15%) 
     + 商圈类型(25%) + 成本指数(15%)

评分等级：
  ≥80分：强烈推荐
  60-79分：值得考虑
  40-59分：需谨慎评估
  <40分：不建议
```

### 2. 品类结构标准模型
```
便利店标准SKU结构（2000-3000个）：
  鲜食/速食：20-30%（高毛利40%+）
  酒水饮料：15-20%（毛利25%）
  休闲零食：10-15%（毛利30%）
  粮油调味：5-10%（毛利20%）
  日化清洁：10-15%（毛利25%）
  日用百货：10-15%（毛利30%）
  个护美妆：3-5%（毛利35%）
  其他：5-10%

商圈类型调整：
  住宅区 → 提升日百/粮油占比
  写字楼 → 提升鲜食/饮料/咖啡占比
  学校   → 提升零食/饮料/文具占比
  交通枢纽 → 提升饮料/旅行用品占比
```

### 3. 定价规则引擎
```
定价 = f(成本, 品类毛利目标, 价格敏感度, 竞品价格)

规则：
  1. 高敏感品（饮料/牛奶/鸡蛋）→ 竞争导向，≤竞品价格
  2. 低敏感品（日百/个护）→ 成本加成，毛利30-50%
  3. 鲜食 → 价值定价，毛利40-60%
  4. 引流品 → ≤1元或≤成本价
  5. 尾部清仓 → 阶梯降价，保本即出
```

## 六、交付方式

每个能力模块作为一个**独立skill**，通过统一入口调度：

```
store-consultant/          # 主skill
  ├── SKILL.md             # 路由入口
  ├──选址评估.md            # 能力一
  ├──选品规划.md            # 能力二  
  ├──定价策略.md            # 能力三
  ├──运营诊断.md            # 能力四
  └──持续辅导.md            # 能力五
```

用户只需说"帮我评估这个地址开店"或"帮我规划选品"，智能体自动路由到对应能力模块。
