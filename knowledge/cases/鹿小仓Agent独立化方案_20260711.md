# 鹿小仓Agent独立化方案 — 从QClaw寄生到自主可控

## 问题本质

QClaw/OpenClaw是开发环境，不是交付环境。我们需要把Agent从"QClaw里的workspace"变成"独立可部署的资产"。

## 当前状态盘点

### 已经有的独立基础
- `C:\Users\13522\diancanmou\` — 已有独立FastAPI应用骨架（app.py + SQLite + 前端页面）
- 已有独立大模型API调用（不经过QClaw）
- 已有独立配置体系（config.ini）
- 桌面已有启动快捷方式（启动鹿小仓.bat）

### 但问题是——这个"独立版"目前是个空壳
对照教程四模块，现状如下：

| 模块 | QClaw里的版本（丰富） | diancanmou独立版（贫瘠） |
|------|----------------------|------------------------|
| 记忆体系 | LCM+MEMORY.md+memory日志+gbrain | ❌ 只有SQLite存对话记录，无长期记忆 |
| Agentic Loop | OpenClaw每轮自动跑 | ❌ 单次请求-响应，没有自主循环 |
| 工具调用 | exec/read/write/web/browser等20+工具 | ❌ 只有LLM对话，不能执行任何实际操作 |
| 知识资产 | 门店档案/竞品档案/案例库/架构蓝图 | ❌ SYSTEM_PROMPT里硬编码了几行摘要 |
| 业务脚本 | 30+个Python脚本（比价/采集/分析） | ❌ scripts目录是空的 |
| 多端部署 | 无 | ❌ 只能本机localhost访问 |

**结论：diancanmou目前只是一个"聊天网页"，不是Agent。它没有Loop、没有工具、没有记忆、没有知识库。**

给diancanmou装上教程说的四层记忆。

技术选型：
- 向量库：**sqlite-vec**（SQLite扩展，不需要额外部署服务）
- 向量化模型：**bge-small-zh**（本地运行，不依赖API）
- 记忆压缩：任务结束后调用LLM生成摘要

### 阶段3：Agentic Loop引擎（3天）
这是最关键的——让Agent能自主循环执行任务。

```python
# 核心Loop逻辑（要写的）
class AgentLoop:
    def __init__(self, llm_client, tools, memory):
        self.llm = llm_client
        self.tools = tools  # 工具注册表
        self.memory = memory
    
    async def run(self, user_goal):
        """主循环：Observe→Think→Act→Receive→Reflect"""
        context = self.memory.recall(user_goal)  # 召回记忆
        tasks = self.llm.plan(user_goal, context)  # 拆解任务
        
        for task in tasks:
            # Think: 决定调用什么工具
            action = self.llm.decide(task, context)
            # Act: 执行工具
            result = self.tools.execute(action)
            # Receive: 结果进入上下文
            context.add(result)
            # Reflect: 检查是否完成
            if self.llm.is_complete(task, result):
                break
        
        # 归档记忆
        self.memory.archive(user_goal, context)
```

工具注册表（第一阶段先实现5个核心工具）：
1. `read_excel` — 读取Excel数据
2. `write_excel` — 写入Excel输出
3. `web_search` — 网页搜索
4. `poi_scan` — 高德API商圈扫描
5. `price_compare` — 商品比价

## 关键决策点（需要你拍板）

1. **手机端用什么？** 微信小程序 vs PWA vs 跨平台App
2. **部署在哪？** 本地PC（仅内网用） vs 云服务器（可外网访问，门店手机能连）
3. **大模型用什么？** 继续用火山引擎（豆包） vs 换智谱/DeepSeek/本地部署
4. **先做哪一步？** 知识迁移（最快见效） vs Loop引擎（最核心） vs 手机端（最吸引人但最远）
